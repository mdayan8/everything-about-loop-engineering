# An Honest Note About This Repository

> **This repo documents a discipline that is approximately two weeks old as of mid-June 2026.** Treat everything here as a snapshot, not a final answer. It will need updates as the tools, practices, and consensus evolve.

## What This Repo Is

A learning resource and reference guide for **loop engineering** — the practice of designing systems that prompt AI coding agents instead of prompting them yourself. It compiles public statements, published essays, open-source tooling, and community discussion into a structured course.

## What This Repo Is Not

- **It is not an established field with years of peer-reviewed best practice.** Loop engineering as a named discipline was coined by Addy Osmani on June 8, 2026, building on work by Boris Cherny, Peter Steinberger, and Geoffrey Huntley. The entire history spans roughly 11 months.
- **It is not a guarantee of results.** Loops can fail, waste tokens, introduce bugs, or ship insecure code. Every cautionary example in this repo is real or drawn from real reports.
- **It is not the final word.** Tools change, patterns evolve, and what counts as "best practice" today may be outdated in months.

## Claim Classification

Every factual claim in this repo falls into one of three categories:

1. **Verified** — sourced from published writing, public talks, or official documentation. Attributed inline.
2. **Self-reported** — described by an individual (e.g., on social media or a blog) but not independently confirmed. Always marked as "self-reported."
3. **Community-reported** — drawn from public discussion threads (Hacker News, GitHub issues, etc.). Attributed to the thread, not treated as authoritative.

If you find an unclassified claim, please [open an issue](../../issues).

## Disambiguation

"Loop engineering" is also an unrelated, older term in structural biology — referring to the re-engineering of flexible loop regions in enzymes like Cas9. That field has nothing to do with this repository. If you arrived here from a biology search result, this is the wrong repo.

## How to Use This Repo

1. Read this file first.
2. Start at [Module 00: Prerequisites](modules/00-prerequisites/README.md).
3. Treat templates as starting points, not production configs.
4. Treat examples as illustrations, not deployments.
5. When in doubt, keep your loop at L1 (report only) longer than feels necessary.

## A Living Document

This repo will evolve. As the community builds more experience with loops, patterns will be refined, new failure modes will emerge, and tools will change. Contributions of corrections, real-world experience reports, and new patterns are welcome — see [CONTRIBUTING.md](CONTRIBUTING.md).
